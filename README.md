# UK Training Provider Scraper

A reusable Python project for collecting publicly available business contact data from UK training-provider directories. It provides both a command-line interface and a simple Windows GUI, with CSV and XLSX exports.

The currently supported directory is the **CITB Approved Training Organisation directory**. Other directory sources require separate adapters and are not implemented yet.

## Features

- Requests/BeautifulSoup-based CITB scraper; no browser automation required.
- CSV and formatted XLSX export.
- Provider deduplication and normalized fields.
- Public business-email extraction from CITB.
- Optional, bounded discovery of missing emails on published provider websites.
- Address and town/location extraction from CITB data.
- Configurable rate limiting, retries, and request timeouts.
- Live Windows GUI with progress, results, counters, and output-folder access.
- Reusable adapter structure for adding future directories.

## Supported directory

| Directory | Status |
| --- | --- |
| CITB Approved Training Organisations | Implemented |

Only CITB is supported in version 1.0.0.

## Output fields

- `provider_name`
- `town_or_location`
- `address`
- `website`
- `email`
- `phone`
- `source_url`
- `email_source`
- `notes`

Small example exports are included in [`samples/`](samples/). Files generated by local runs are saved in `outputs/` and ignored by Git.

## Windows quick start

1. Extract the ZIP or clone this repository.
2. Double-click `install.bat` once. It creates a private `.venv` and installs the dependencies.
3. Double-click `run_gui.bat`.
4. Select CITB, choose the email option and output name, then enter a Limit or leave Limit empty to scrape all providers.
5. Click **Start**. Results are saved in `outputs/`.

The GUI remains responsive while scraping and displays completed providers and field counters as they arrive.

### Email behaviour

Email addresses published directly by CITB are always exported. The **Search missing emails on provider websites** checkbox controls only the additional step that checks a small number of public pages on a provider's published website when the CITB email is missing.

## Developer setup

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

## CLI usage

Five-provider CITB test without website email discovery:

```powershell
python run.py --scraper citb --limit 5 --no-email-finder --output-name test_5
```

Fifty-provider run with missing-email discovery enabled:

```powershell
python run.py --scraper citb --limit 50 --output-name citb_50
```

Full CITB directory without website email discovery:

```powershell
python run.py --scraper citb --no-email-finder --output-name citb_full
```

For a quick batch-file example, run `run_cli_example.bat` after installation.

## Project structure

```text
core/                 Shared HTTP, export, email, and normalization logic
scrapers/             Directory-specific adapters
outputs/              Locally generated exports (Git-ignored)
samples/              Small example CSV/XLSX files
app.py                Shared application service for CLI and GUI
run.py                Command-line entry point
gui.py / gui.pyw      Windows GUI and no-console launcher
install.bat           One-time Windows setup
run_gui.bat           Windows GUI launcher
run_cli_example.bat   Small CLI example
config.yaml           Timeouts, rate limit, output, and logging settings
```

## Adding a new directory adapter

1. Create or copy a scraper module under `scrapers/`.
2. Keep directory-specific endpoints, selectors, and parsing rules inside that adapter.
3. Return the same normalized provider fields listed above.
4. Reuse `core/exporter.py`, `core/email_finder.py`, and `core/normalizer.py` instead of duplicating shared logic.
5. Register the adapter in `run.py` and `app.py` where needed, including the `--scraper` choices.
6. Add a small sample and tests for the new adapter before describing it as supported.

## Ethical scraping

- Collects only publicly available business information.
- Does not bypass login screens, CAPTCHAs, paywalls, or anti-bot controls.
- Uses rate limiting, retries, request timeouts, and a descriptive user agent.
- Keeps website email discovery bounded to a small number of same-domain pages.
- Site operators' terms and applicable laws remain the user's responsibility.

## Limitations

- Website is exported only when CITB explicitly provides one; it is never invented from an email domain.
- Some providers may not expose every optional field.
- Directory markup and endpoints can change and may require adapter maintenance.
- Future training directories require new adapters; they are not included in this release.

## Local running costs

There are no ongoing software or subscription costs for local use. Python and the included dependencies are open source. Normal internet access is required to install dependencies and retrieve public directory data.
